# 🏃‍♀️ Runner's High 사용자 모델링 및 요구사항 정의

## 📌 서비스 개요
**Runner's High**는 러닝 데이터를 시각화하고 동기부여를 통해 꾸준한 운동을 유도하는 러너 맞춤형 트래킹 앱입니다.

---

## 📌 사용자는 누구인가?

| 사용자 유형 | 정의 |
|-------------|------|
| 건강 관리형 러너 | 운동을 통해 체력 향상과 스트레스 해소를 목표로 하는 일반 사용자 |
| 데이터 애호형 러너 | 자신의 운동 데이터를 체계적으로 기록하고 분석하길 원하는 사용자 |
| 꾸준함 추구형 러너 | 지속적인 목표 설정과 동기부여로 운동 습관을 형성하려는 사용자 |

## 📌 사용자 시나리오

### 🎯 목표: 무리하지 않고, 체력에 맞는 운동 루틴을 설정하고 꾸준히 달리기

**시나리오**  
정다은은 하루 일과를 마치고, 앱을 켜서 이번 주 달성률을 확인합니다.  
아직 주간 목표까지 2km가 남았고, 앱은 오늘 3km 경로를 추천합니다.  
“러닝 시작” 버튼을 누르고 실시간 페이스, 거리, 시간을 확인하며 달립니다.  
운동을 마치고 나면 앱이 자동으로 GPS 경로와 구간별 페이스를 분석하고  
“24시간 회복 권장”이라는 피드백과 함께 새로운 배지를 부여합니다.
또한 러닝화의 상태를 총 누적거리로 확인하고 새로운 러닝화를 살 계획을 세웁니다.
이전에 수명이 다한 러닝화 목록을 통해 성취감을 느낍니다.
정다은은 레벨이 올라간 것을 확인하고 뿌듯함을 느낍니다.

## 📌 사용자 스토리 및 인수 조건

### 1. 주간 목표 기능

**사용자 스토리**  
“사용자로서, 저는 주간 목표를 확인하고 싶습니다.  
그래야 얼마나 더 운동해야 하는지 알 수 있습니다.”

**인수 조건**
- **Given**: 사용자가 대시보드 화면을 보고 있을 때
- **When**: ‘주간 목표’ 섹션을 확인할 경우
- **Then**: 누적 거리와 총 목표, 퍼센트바가 표시되어야 한다

---

### 2. 실시간 러닝 기록 기능

**사용자 스토리**  
“사용자로서, 러닝 중 실시간 거리와 페이스를 보고 싶습니다.  
그래야 내 운동 강도를 조절할 수 있습니다.”

**인수 조건**
- **Given**: 사용자가 '실시간 러닝' 화면에서 러닝을 시작했을 때
- **When**: 러닝이 진행 중일 때
- **Then**: 거리, 페이스, 시간 정보가 1초 간격으로 갱신되어야 한다

---

### 3. 운동 후 리포트 및 피드백

**사용자 스토리**  
“사용자로서, 운동 후 나의 운동 리포트를 보고 싶습니다.  
그래야 내가 잘 달렸는지, 얼마나 회복해야 할지 알 수 있습니다.”

**인수 조건**
- **Given**: 사용자가 러닝을 종료하고 리포트를 열었을 때
- **When**: 최근 기록을 선택했을 때
- **Then**: 경로, 페이스, 시간, 고도, 심박수 등이 시각화되어야 한다
- **And**: 회복 권장 메시지가 함께 표시되어야 한다

---

### 4. 배지 및 대표 배지 기능

**사용자 스토리**
    “사용자로서, 대표 배지를 설정하고 싶습니다. 그래야 내 성취를 다른 사용자에게 보여줄 수 있습니다.”

**인수 조건**
- **Given**: 사용자가 '챌린지' 화면을 보고 있을 때
- **When**: 배지를 클릭했을 때
- **Then**: 선택한 배지가 대표 배지로 설정되어야 하며, 순위표에 함께 표시되어야 한다

---

### 5. 러닝화 관리 기능

**사용자 스토리**
    “사용자로서, 현재 사용 중인 러닝화의 누적 거리를 확인하고 싶습니다. 그래야 교체 시점을 알 수 있습니다.”

**인수 조건**
- **Given**: 사용자가 대시보드 내 '나의 러닝화' 섹션을 볼 때
- **When**: 신발 카드가 나타날 때
- **Then**: 누적 거리와 은퇴 버튼이 표시되어야 한다

---

### 6. 인기 코스 시각화 기능

**사용자 스토리**
    “사용자로서, 다른 사람들이 자주 달린 코스를 보고 싶습니다. 그래야 나도 따라 달릴 수 있습니다.”

**인수 조건**
- **Given**: 사용자가 '인기 코스' 섹션으로 이동했을 때
- **When**: 지도가 렌더링될 때
- **Then**: 여러 경로가 겹치는 구간이 진하게 표시되어야 한다

---

### 7. 챌린지 순위 기능

**사용자 스토리**
    “사용자로서, 친구들과의 주간 거리 순위를 보고 싶습니다. 그래야 동기부여를 받을 수 있습니다.”

**인수 조건**
- **Given**: 사용자가 '챌린지' 섹션을 볼 때
- **When**: 친구 또는 지역 러너 탭을 선택했을 때
- **Then**: 주간 거리 기준으로 정렬된 순위표가 표시되어야 한다

---

### 8. 레벨별 테마 변경 기능

***사용자 스토리***
    "달린 누적 거리가 늘어날 수록, 레벨별 색상 테마 변경을 통해 동기부여를 지속할 수 있습니다."

**인수 조건**
- **Given**: 사용자가 앱에 접속할때
- **When**: 150km 단위의 누적거리별로 레벨이 변화한다.
- **Then**: 사용자의 레벨에따라 테마가 변경된다.

---

### 9. 날씨 및 미세먼지 정보 표시 기능

**사용자 스토리**  
“사용자로서, 러닝 시작 전 현재 날씨와 미세먼지 상태를 알고 싶습니다.  
그래야 오늘 러닝을 해도 괜찮은지 판단할 수 있습니다.”

**인수 조건**
- **Given**: 사용자가 ‘실시간 러닝’ 탭을 열었을 때
- **When**: 앱이 위치 기반 날씨 API에 요청을 보냈을 때
- **Then**: 날씨(☀️/☁️/🌧️ 등)와 기온, 미세먼지 상태(😊/😐/😷)가 표시되어야 한다
